Introdução

Sistemas lineares de equações algébricas constituem um dos pilares da modelagem numérica em Física e Engenharia. Na maioria dos problemas reais, as equações diferenciais que descrevem fenômenos físicos não são resolvidas diretamente em sua forma contínua. Em vez disso, procedimentos de discretização — como diferenças finitas, elementos finitos ou volumes finitos — transformam o problema contínuo em um sistema linear da forma:

\[A \mathbf{x} = \mathbf{b}\]

Nesse contexto, a matriz \(A\) incorpora as propriedades geométricas, físicas e constitutivas do sistema. O vetor \(\mathbf{x}\) representa as incógnitas discretizadas (como deslocamentos, pressões ou temperaturas), enquanto o vetor \(\mathbf{b}\) reúne as fontes externas e as condições de contorno.

Relevância na Engenharia

Na engenharia aeronáutica, por exemplo, esses sistemas são onipresentes: * Análise Estrutural: O Método dos Elementos Finitos (MEF) aplicado a asas e fuselagens gera sistemas gigantescos onde \(A\) é a matriz de rigidez. * Dinâmica de Fluidos (CFD): A discretização das equações de Navier-Stokes gera sistemas lineares a cada passo de tempo, cujo custo de resolução domina o tempo total de simulação.

O Desafio Computacional

Sistemas com milhares ou milhões de graus de liberdade tornam inviável o uso “ingênuo” de métodos diretos simples. É necessário explorar a estrutura da matriz (como a esparsidade e a simetria) e analisar o condicionamento do sistema para garantir que erros numéricos não destruam a solução.

O estudo desses métodos não é apenas uma questão algorítmica, mas uma compreensão de como a física do problema dita a matemática da solução. Este capítulo apresentará os métodos diretos e iterativos, analisando seus fundamentos, custo computacional e aplicabilidade prática.

Formulação Geral dos Sistemas Lineares

Um sistema linear de \(n\) equações e \(n\) incógnitas pode ser escrito de forma compacta como: \[A \mathbf{x} = \mathbf{b}\]

Onde: * \(A \in \mathbb{R}^{n \times n}\) é a matriz dos coeficientes. * \(\mathbf{x} \in \mathbb{R}^n\) é o vetor das incógnitas. * \(\mathbf{b} \in \mathbb{R}^n\) é o vetor dos termos independentes.

Em forma expandida, o sistema assume a forma: \[\sum_{j=1}^n A_{ij} x_j = b_i, \quad i = 1, \dots, n\]

Do ponto de vista geométrico, cada equação representa um hiperplano em \(\mathbb{R}^n\), e a solução do sistema corresponde ao ponto de interseção desses hiperplanos. Quando \(A\) é inversível (\(\det(A) \neq 0\)), essa interseção é única e o sistema admite uma solução bem definida.

Existência, Unicidade e Condicionamento

A existência e unicidade da solução dependem das propriedades da matriz \(A\). O sistema possui solução única se, e somente se, \(\det(A) \neq 0\). Equivalentemente, \(A\) deve ser uma matriz de posto completo (\(\text{rank}(A) = n\)).

Quando essas condições não são satisfeitas, o sistema pode apresentar infinitas soluções ou nenhuma solução, o que, fisicamente, indica um problema mal formulado, como ausência de condições de contorno suficientes.

Sensibilidade Numérica e o Número de Condicionamento

Mesmo quando \(\det(A) \neq 0\), o sistema pode ser numericamente instável. O número de condicionamento \(\kappa(A)\) mede a sensibilidade da solução \(\mathbf{x}\) a pequenas perturbações nos dados de entrada (\(A\) ou \(\mathbf{b}\)): \[\kappa(A) = \|A\| \cdot \|A^{-1}\|\]

Valores elevados de \(\kappa(A)\) indicam que o sistema é “mal condicionado”, ou seja, erros mínimos de arredondamento podem ser amplificados drasticamente na solução final.

Exemplo Prático de Mau Condicionamento

Considere o sistema quase singular abaixo: \[ \begin{cases} 1.0000x + 1.0000y = 2.0000 \\ 1.0000x + 1.0001y = 2.0001 \end{cases} \implies \mathbf{x} = \begin{bmatrix} 1 \\ 1 \end{bmatrix} \]

Se alterarmos o termo \(b_2\) em apenas \(0.005\%\) (de \(2.0001\) para \(2.0002\)): \[ \begin{cases} 1.0000x + 1.0000y = 2.0000 \\ 1.0000x + 1.0001y = 2.0002 \end{cases} \implies \mathbf{x} = \begin{bmatrix} 0 \\ 2 \end{bmatrix} \] A solução variou 100% devido a uma perturbação ínfima, ilustrando o perigo de matrizes com \(\kappa(A)\) elevado.

Estrutura das Matrizes em Problemas Físicos

Na engenharia, a matriz \(A\) frequentemente possui propriedades que permitem otimizar sua resolução:

  • Matrizes Simétricas (\(A = A^T\)): Comuns em problemas de elasticidade linear e princípios variacionais.
  • Matrizes Definidas Positivas: Para todo vetor \(\mathbf{v} \neq 0\), temos \(\mathbf{v}^T A \mathbf{v} > 0\). Isso garante estabilidade para diversos métodos diretos.
  • Matrizes Esparsas: A maioria dos coeficientes é nula. Em malhas de Elementos Finitos, a esparsidade é a regra, permitindo economizar memória e tempo de CPU.
  • Matrizes Banda: Os elementos não nulos estão concentrados próximos à diagonal principal.

Metodos Diretos para Sistemas Lineares

Os metodos diretos visam obter a solucao exata do sistema linear \(A x = b\) por meio de um numero finito de operacoes algebricas. Na pratica computacional, a solucao obtida e afetada apenas por erros de arredondamento de maquina.

O metodo direto mais fundamental e a Eliminacao de Gauss, da qual derivam quase todos os algoritmos modernos de resolucao de sistemas lineares.

Eliminacao de Gauss

A eliminacao de Gauss consiste em transformar o sistema original em um sistema equivalente de forma triangular superior, permitindo a solucao por substituicao retroativa.

Considere o sistema \(A x = b\). O objetivo e anular os coeficientes abaixo da diagonal principal, isto e, \(a_{ij} = 0\) para \(i > j\).

1. Etapa de Eliminacao

Na etapa \(k\), elimina-se o termo \(a_{ik}\) para todo \(i > k\), utilizando multiplicadores:

\(m_{ik} = a_{ik} / a_{kk}\)

As linhas sao atualizadas segundo a regra:

\(L_i = L_i - m_{ik} L_k\)

Para cada linha \(i\) de \(k+1\) ate \(n\). Apos \(n-1\) etapas, o sistema assume a forma \(U x = c\), onde \(U\) e uma matriz triangular superior.

2. Substituicao Retroativa

A solucao e obtida resolvendo o sistema de baixo para cima:

\(x_n = c_n / u_{nn}\)

\(x_i = (1 / u_{ii}) (c_i - \sum_{j=i+1}^n u_{ij} x_j)\)

Para \(i = n-1, ..., 1\).

Decomposicao LU e Pivotamento

Matricialmente, a eliminacao de Gauss e equivalente a fatoracao \(A = L U\), onde \(L\) e uma matriz triangular inferior e \(U\) e uma matriz triangular superior.

Pivotamento e Estabilidade Numerica

Para evitar divisoes por zero ou por numeros muito pequenos que degradam a precisao, utiliza-se o Pivotamento Parcial. O algoritmo seleciona o elemento de maior valor absoluto na coluna atual:

\(|a_{pk}| = max |a_{ik}|\) para \(i \ge k\)

Essa operacao resulta na fatoracao com permutacao:

\(P A = L U\)

Onde \(P\) e a matriz que registra as trocas de linhas realizadas para manter a estabilidade.

Code 
using LinearAlgebra

# 1. Definição do sistema Ax = b
A = [1.0  2.0  3.0;
     2.0  1.0  2.0;
     3.0  4.0  1.0]

b = [14.0, 10.0, 14.0]
n = length(b)

# --- FASE 1: ELIMINAÇÃO ---
U = copy(A)
c = copy(b)

for k in 1:n-1
    for i in k+1:n
        m = U[i,k] / U[k,k]
        U[i, k:n] .-= m .* U[k, k:n]
        c[i] -= m * c[k]
    end
end

# --- FASE 2: SUBSTITUIÇÃO RETROATIVA ---
x = zeros(n)
for i in n:-1:1
    soma = sum(U[i, i+1:n] .* x[i+1:n])
    x[i] = (c[i] - soma) / U[i,i]
end

println("Solução x encontrada:")
display(x)

# --- FASE 3: VERIFICAÇÃO E NORMAS ---

# Cálculo do resíduo: r = b - Ax
resíduo = b - (A * x)

# Cálculo da Norma Euclidiana (Norma L2)
# Representa a "magnitude" do erro global
erro_norma = norm(resíduo)

println("\nVetor Resíduo (b - Ax):")
display(resíduo)

println("\nNorma do Resíduo: ", erro_norma)

# Comparação com a precisão da máquina (eps)
if erro_norma < 1e-14
    println("Status: Solução converge com sucesso (Erro < 10^-14)")
else
    println("Status: Alerta de possível instabilidade numérica")
end

Metodos Iterativos para Sistemas Lineares

Diferentemente dos metodos diretos, os metodos iterativos constroem uma sequencia de aproximacoes \(x^{(0)}, x^{(1)}, x^{(2)}, \dots\) que converge para a solucao exata do sistema \(Ax = b\). A essencia destes metodos e transformar o sistema linear em um problema de ponto fixo:

\(x^{(k+1)} = T x^{(k)} + c\)

Onde \(T\) e a matriz de iteracao e \(c\) e um vetor constante.

Demonstracao da Condicao de Convergencia

Para que o metodo funcione, o erro deve diminuir a cada passo. Definimos o erro na iteracao \(k\) como \(e^{(k)} = x^{(k)} - x\), onde \(x\) e a solucao exata. Subtraindo a solucao exata (\(x = Tx + c\)) da formula de iteracao:

\(x^{(k+1)} - x = (T x^{(k)} + c) - (T x + c)\) \(e^{(k+1)} = T e^{(k)}\)

Aplicando recursivamente desde a iteracao inicial \(e^{(0)}\): \(e^{(k)} = T^k e^{(0)}\)

Para que \(e^{(k)} \to 0\) para qualquer erro inicial, e necessario e suficiente que o raio espectral de \(T\) (o maior autovalor em modulo) seja menor que um: \(\rho(T) < 1\)

1. Metodo de Jacobi

O Metodo de Jacobi isola cada variavel \(x_i\) utilizando apenas os valores da iteracao anterior. Matematicamente, decompomos \(A = D + L + U\), onde \(D\) e a diagonal, \(L\) e a triangular inferior e \(U\) a superior (ambas com diagonal nula).

O sistema \(Ax = b\) torna-se: \((D + L + U)x = b\) \(Dx = b - (L + U)x\) \(x = D^{-1}(b - (L + U)x)\)

Assim, a matriz de iteracao de Jacobi e \(T_J = -D^{-1}(L + U)\).

Forma por Componentes

Para cada linha \(i = 1, \dots, n\): \(x_i^{(k+1)} = (1 / a_{ii}) (b_i - \sum_{j \ne i} a_{ij} x_j^{(k)})\)

2. Metodo de Gauss-Seidel

O metodo de Gauss-Seidel acelera a convergencia ao utilizar os valores de \(x\) assim que eles sao calculados. Na decomposicao matricial, mantemos a parte inferior no lado esquerdo:

\((D + L)x = b - Ux\) \(x = (D + L)^{-1} b - (D + L)^{-1} Ux\)

A matriz de iteracao de Gauss-Seidel e \(T_{GS} = -(D + L)^{-1}U\).

Forma por Componentes

\(x_i^{(k+1)} = (1 / a_{ii}) (b_i - \sum_{j < i} a_{ij} x_j^{(k+1)} - \sum_{j > i} a_{ij} x_j^{(k)})\)

Note que para \(j < i\), o algoritmo ja utiliza o valor “novo”, o que geralmente dobra a velocidade de convergencia em relacao ao Jacobi.

Criterio de Convergencia: Dominancia Diagonal

Uma condicao suficiente para a convergencia de ambos os metodos e que a matriz \(A\) seja estritamente diagonal dominante: \(|a_{ii}| > \sum_{j \ne i} |a_{ij}|\) para todo \(i\).

Fisicamente, isso significa que a influencia da variavel sobre sua propria equacao e maior que a soma das influencias das outras variaveis, garantindo estabilidade no processo de relaxacao.

Code 
using LinearAlgebra
using Printf

# 1. Definição do Sistema (Diagonal Dominante)
A = [10.0 -1.0  2.0;
     -1.0 11.0 -1.0;
      2.0 -1.0 10.0]

b = [6.0, 25.0, -11.0]

# --- FUNÇÃO AUXILIAR PARA CÁLCULO DE RESÍDUO ---
# r = ||b - Ax||
calcular_residuo(A, b, x) = norm(b - A * x)

# --- MÉTODO DIRETO: GAUSS ---
function eliminacao_gauss(A_orig, b_orig)
    U, c = copy(A_orig), copy(b_orig)
    n = length(c)
    for k in 1:n-1
        for i in k+1:n
            m = U[i,k] / U[k,k]
            U[i, k:n] .-= m .* U[k, k:n]
            c[i] -= m * c[k]
        end
    end
    x = zeros(n)
    for i in n:-1:1
        soma = sum(U[i, i+1:n] .* x[i+1:n])
        x[i] = (c[i] - soma) / U[i,i]
    end
    return x
end

# --- MÉTODOS ITERATIVOS ---
function resolver_iterativo(A, b; metodo=:gauss_seidel, tol=1e-8, max_iter=100)
    n = length(b)
    x = zeros(n)
    x_old = zeros(n)
    for k in 1:max_iter
        x_old .= x
        for i in 1:n
            soma = 0.0
            for j in 1:n
                if i != j
                    val_x = (metodo == :jacobi) ? x_old[j] : x[j]
                    soma += A[i,j] * val_x
                end
            end
            x[i] = (b[i] - soma) / A[i,i]
        end
        if norm(x - x_old, Inf) < tol
            return x, k
        end
    end
    return x, max_iter
end

# --- EXECUÇÃO E COMPARAÇÃO ---
x_gauss = eliminacao_gauss(A, b)
x_jacobi, it_j = resolver_iterativo(A, b, metodo=:jacobi)
x_gs, it_gs = resolver_iterativo(A, b, metodo=:gauss_seidel)

# --- TABELA DE RESULTADOS ---
println("---------------------------------------------------------")
println("MÉTODO         | ITERAÇÕES | NORMA DO RESÍDUO (||b - Ax||)")
println("---------------------------------------------------------")
@printf("Direto (Gauss) | %-9s | %.16e\n", "N/A", calcular_residuo(A, b, x_gauss))
@printf("Jacobi         | %-9d | %.16e\n", it_j, calcular_residuo(A, b, x_jacobi))
@printf("Gauss-Seidel   | %-9d | %.16e\n", it_gs, calcular_residuo(A, b, x_gs))
println("---------------------------------------------------------")

Metodo da Relaxacao Sucessiva (SOR)

O metodo da Relaxacao Sucessiva (Successive Over-Relaxation, SOR) e uma generalizacao do metodo de Gauss-Seidel, introduzindo um parametro de relaxacao \(\omega\) com o objetivo de acelerar a convergencia do processo iterativo.

Partindo da decomposicao \(A = D + L + U\), o metodo de Gauss-Seidel e um caso particular do SOR com \(\omega = 1\). No metodo SOR, a iteracao e definida como:

\(( D + \omega L ) x^{(k+1)} = (1 - \omega) D x^{(k)} - \omega U x^{(k)} + \omega b\)

Forma Componente do Metodo SOR Em termos componentes, a iteracao assume a forma:

\(x_i^{(k+1)} = (1 - \omega) x_i^{(k)} + \frac{\omega}{a_{ii}} ( b_i - \sum_{j < i} a_{ij} x_j^{(k+1)} - \sum_{j > i} a_{ij} x_j^{(k)} )\)

O parametro \(\omega\) controla a convergencia: * \(0 < \omega < 1\): Sub-relaxacao (usada para estabilizar sistemas divergentes). * \(\omega = 1\): Metodo de Gauss-Seidel. * \(1 < \omega < 2\): Sobre-relaxacao (acelera a convergencia em sistemas estaveis).

Abaixo, comparamos como a escolha do \(\omega\) impacta o numero de iteracoes.

Code 
using LinearAlgebra
using Printf

# 1. Definição do Sistema (Matriz de Teste)
A = [4.0 -1.0 0.0; 
    -1.0 4.0 -1.0; 
     0.0 -1.0 4.0]
b = [15.0, 10.0, 10.0]

# 2. Função SOR (que engloba Gauss-Seidel se omega=1.0)
function resolver_com_analise(A, b, omega; tol=1e-8, max_iter=500)
    n = length(b)
    x = zeros(n)
    
    for k in 1:max_iter
        x_old = copy(x)
        for i in 1:n
            soma = 0.0
            for j in 1:n
                if i != j
                    soma += A[i,j] * x[j]
                end
            end
            # Equação característica do SOR
            x[i] = (1 - omega) * x[i] + (omega / A[i,i]) * (b[i] - soma)
        end
        
        # Critério de parada: proximidade entre iterações
        if norm(x - x_old, Inf) < tol
            residuo_final = norm(b - A * x)
            return x, k, residuo_final
        end
    end
    return x, max_iter, norm(b - A * x)
end

# 3. Execução dos Casos
sol_gs, it_gs, res_gs   = resolver_com_analise(A, b, 1.0)   # Gauss-Seidel
sol_sor, it_sor, res_sor = resolver_com_analise(A, b, 1.01) # SOR com omega otimizado

# 4. Exibição do Comparativo
println("-"^60)
println(@sprintf("%-15s | %-10s | %-20s", "Método", "Iterações", "Resíduo Final"))
println("-"^60)
@printf("%-15s | %-10d | %.16e\n", "Gauss-Seidel", it_gs, res_gs)
@printf("%-15s | %-10d | %.16e\n", "SOR (w=1.01)", it_sor, res_sor)
println("-"^60)

# Verificação de proximidade entre as soluções
println("\nDiferença entre soluções: ", norm(sol_gs - sol_sor))
